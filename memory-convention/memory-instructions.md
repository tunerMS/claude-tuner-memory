# Конвенция файловой памяти

Это инструкция, которую стоит положить в свой `~/.claude/CLAUDE.md` (или
project-level `CLAUDE.md`), чтобы Claude вёл персистентную память по единому
формату. memtools/harvester/homunculus работают поверх именно этой структуры.

---

## Memory (вставь в CLAUDE.md)

У тебя есть персистентная файловая память в
`~/.claude/projects/<slug>/memory/`. Каждая память — это один файл с одним
фактом и frontmatter:

```markdown
---
name: <короткий-kebab-case-слаг>
description: <одна строка — по ней решается релевантность при вспоминании>
metadata:
  type: user | feedback | project | reference
---

<сам факт; для feedback/project добавь строки **Почему:** и **Как применять:**.
Связывай родственные заметки через [[их-name]].>
```

В теле ссылайся на родственные заметки через `[[name]]`, где `name` — слаг
другой заметки. Ссылайся щедро — `[[name]]` на ещё несуществующую заметку
нормально: это метка «стоит написать позже».

**Типы:**
- `user` — кто пользователь (роль, экспертиза, предпочтения).
- `feedback` — как тебе работать (правки и подтверждённые подходы); указывай почему.
- `project` — текущая работа, цели, ограничения, не выводимые из кода/гита; относительные даты переводи в абсолютные.
- `reference` — указатели на внешние ресурсы (URL, дашборды, тикеты).

После записи файла добавь однострочный указатель в `MEMORY.md`
(`- [Заголовок](file.md) — хук`). `MEMORY.md` — это всегда-загружаемый индекс,
по одной строке на заметку, без frontmatter.

Перед сохранением проверь, нет ли уже файла на эту тему — обнови его, а не плоди
дубликаты; удаляй заметки, оказавшиеся неверными. Не сохраняй то, что и так есть
в репозитории (структура кода, прошлые фиксы, история гита, CLAUDE.md) или важно
лишь для текущего диалога.

---

## Почему именно так

- **Один факт = один файл** → точечные правки, чистые диффы, не растёт монолит.
- **MEMORY.md тонкий** → всегда-загружаемый контекст не пухнет; детали тянутся по требованию (`mem recall`).
- **frontmatter `name`/`description`** → линковка `[[ ]]` и семантический отбор работают предсказуемо.
- **`[[links]]`** → memtools-линкер достраивает граф связей; harvester и graphify тоже опираются на эту структуру.
