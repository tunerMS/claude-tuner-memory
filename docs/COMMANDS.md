# Справочник команд

Все команды, которыми управляется этот сетап: CLI memtools, слэш-команды для инстинктов и служебные скрипты. По-русски, как и остальные helper-доки; основной README — английский.

## `mem` — CLI семантического слоя (memtools)

Обёртка `~/.claude/memory-tools/mem` сама подхватывает venv и `PYTHONPATH` — зови её напрямую, ничего активировать не нужно.

| Команда | Что делает |
|---|---|
| `mem index` | Строит/обновляет семантический индекс памяти. Инкрементально: по хэшу содержимого перечитываются только изменённые файлы. |
| `mem recall "<запрос>" [-k N]` | Топ-k релевантных чанков по всей памяти (включая огромные `project_*.md` и `sessions/*`) с указанием `файл › заголовок`. Главная команда: вытащить нужный кусок вместо чтения 100-КБ файла целиком. |
| `mem lifecycle` | Пишет `_review.md`: заметки без правок дольше 30 дней + подозрительно похожие пары (почти-дубли). Ничего не удаляет — только кандидаты на ревизию. |
| `mem link` | Поддерживает блок «Related (auto)» с `[[wikilinks]]` внизу каждой курируемой заметки. Аддитивно и идемпотентно. |
| `mem graphify [--deep]` | Выгружает память в граф знаний [graphify](https://github.com/). Стоит LLM-токенов, поэтому только вручную и только при наличии ключа в `graphify.env`. |
| `mem maintain` | `index` + `lifecycle` + `link` одной командой — ровно то, что launchd-агент гоняет каждые 6 часов. |

Пороги и пути настраиваются env-переменными — таблица в [README](../README.md#configuration).

## Слэш-команды для инстинктов (в Claude Code)

Эти команды ставятся стоковым скиллом **continuous-learning-v2** (экосистема ECC) — в этот репозиторий они не входят, но работают поверх тех же файлов `~/.claude/homunculus/instincts/personal/*.md`, которые пишет наш `analyze.sh`.

| Команда | Что делает |
|---|---|
| `/instinct-status` | Обзор всех выученных инстинктов: уверенность, домен, триггер. Флаги: `--domain <имя>`, `--low-confidence`, `--high-confidence`, `--json`. |
| `/evolve` | Кластеризация: 3+ родственных инстинкта → предложение вырастить из них команду, скилл или агента (результат в `~/.claude/homunculus/evolved/`). По умолчанию — превью; `--execute` создаёт файлы, ещё есть `--domain`, `--threshold <n>`, `--type <command\|skill\|agent>`. |
| `/instinct-export` | Выгрузка инстинктов в файл — поделиться с коллегой, перенести на другую машину. |
| `/instinct-import <file>` | Импорт чужих инстинктов (складываются отдельно, в `instincts/inherited/`). |
| `/learn` | Ручной вариант обучения: проанализировать ТЕКУЩУЮ сессию и сохранить стоящие паттерны (из v1-системы; фоновый homunculus делает то же самое автоматически и гранулярнее). |

**Нюанс совместимости.** Скриптовая часть этих команд (`instinct-cli.py` из стокового скилла) читает только `*.yaml`, а этот сетап пишет инстинкты в `*.md` — CLI их не увидит. На практике команды всё равно работают: их тела велят Claude прочитать файлы инстинктов напрямую, а формат (frontmatter `id/trigger/confidence/domain` + `## Action`/`## Evidence`) полностью совпадает.

## Служебные команды (этот репозиторий)

| Команда | Что делает |
|---|---|
| `./install.sh` | Полная установка. Флаги: `--memtools-only` (без harvester/homunculus), `--load-agents` (сразу `launchctl load` фоновых агентов). |
| `./uninstall.sh` | Снимает launchd-агенты и удаляет код подсистем, данные не трогает. `--purge-data` — удалить и данные (наблюдения, инстинкты, логи тест-прогонов; НЕОБРАТИМО). |
| `python3 ~/.claude/harvester/harvester.py <transcript.jsonl>` | Ручной прогон жнеца по одной сессии — посмотреть, как транскрипт сожмётся в session-memory. |
| `python3 ~/.claude/harvester/sweep.py` | Ручной свип: то, что launchd делает каждые ~15 минут (найти отлежавшиеся транскрипты и сжать). |
| `touch ~/.claude/<memory-tools\|harvester\|homunculus>/disabled` | Стоп-кран любой подсистемы; `rm` этого файла — включить обратно. |
| `launchctl load -w / unload ~/Library/LaunchAgents/com.<...>.plist` | Включить/выключить фоновые агенты (`com.memory.maintain`, `com.harvester.sweep`, `com.homunculus.observer`). |

## Чего здесь нет

- `/init` — это встроенная команда Claude Code (генерирует `CLAUDE.md` проекта), к системе памяти не относится.
- Команды `init` нет и в `mem` CLI: первый `mem index` сам создаёт индекс, отдельная инициализация не нужна.
