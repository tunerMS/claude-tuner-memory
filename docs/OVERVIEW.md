# Зачем этот сетап и что он даёт

TL;DR: долговременная память для Claude Code из трёх фоновых подсистем поверх обычных markdown-файлов — семантический поиск по всей памяти, автоматический «дневник проектов» и выучивание твоих привычек. Локально, приватно, почти бесплатно, выключается одной командой.

## Проблема

Из коробки Claude Code помнит ровно то, что лежит в контексте текущей сессии. Штатная файловая память (`CLAUDE.md`, `MEMORY.md`, заметки) — плоские markdown-файлы, и с ростом объёма упираешься в четыре вещи:

1. **Вспоминать = читать целиком.** Ради одного абзаца грузится 100-КБ файл — расход контекста и внимания.
2. **Сессии не связаны.** Новая сессия не знает, чем кончилась вчерашняя; «так, на чём мы остановились?» — каждый раз руками.
3. **Привычки повторяешь заново.** «Ограничивай вывод tail'ом», «коммить атомарно», «пиши тело коммита по-русски» — в каждой новой сессии с нуля.
4. **Заметки гниют.** Протухают, дублируются, теряют связи — руками это никто не чистит.

Memory-платформы (cognee и подобные) это решают, но ценой инфраструктуры (векторная БД + граф), LLM-стоимости на каждый инжест и непрозрачного хранилища. Для персональной памяти, где корпус маленький и ценный, это перебор.

## Что даёт

1. **Точечное вспоминание вместо чтения файлов целиком.** `mem recall "<вопрос>"` достаёт top-k релевантных кусков по смыслу из всей памяти — включая огромные `project_*.md` и журналы сессий — с указанием «файл › заголовок». Эмбеддинги локальные (`multilingual-e5-small`), бесплатно и оффлайн.
2. **«Где мы остановились» подгружается само.** Harvester каждые ~15 минут сжимает завершённые сессии в файл проекта («🔖 текущее состояние» + журнал), а SessionStart-хук вливает этот блок при следующем открытии Claude Code в том же репозитории. Рядом — последние результаты тест-прогонов (gradle/pytest), пойманные из вывода команд.
3. **Claude постепенно начинает работать «по-твоему».** Homunculus наблюдает за сессиями через хуки; паттерны, повторившиеся 3+ раз, дистиллируются в «инстинкты» с уверенностью 0.3–0.9. На старте сессии топ-25 инстинктов с conf ≥ 0.5 вливаются в контекст. Поправил Claude — уверенность инстинкта снизится; подтвердилось ещё раз — вырастет.
4. **Память не гниёт.** `mem lifecycle` помечает протухшее (>30 дней без правок) и почти-дубли в `_review.md` — ничего не удаляя, решение всегда за человеком. `mem link` сам поддерживает `[[wikilinks]]`-связи между родственными заметками.
5. **Канон остаётся простым и твоим.** Всё — обычные markdown-файлы: читаются глазами, диффаются, версионируются git'ом. Ни базы данных, ни облака: три launchd-агента и пара хуков, любая подсистема выключается `touch .../disabled`.

## Как это устроено (в двух словах)

Конвенция «один факт — один файл» + три независимые подсистемы поверх неё:

| Подсистема | Роль | Ритм | Стоимость |
|---|---|---|---|
| **memtools** | семантический индекс, recall, гигиена, связи | каждые 6 ч | 0 — локальные эмбеддинги |
| **harvester** | транскрипт сессии → session-memory проекта | каждые ~15 мин | дешёвый вызов `claude` CLI (на входе маленький дайджест) |
| **homunculus** | наблюдения → инстинкты | каждые 30 мин, только при ≥10 наблюдениях | дешёвый вызов `claude` CLI |

Схема с потоками данных — в [README](../README.md#architecture).

## Принципы

- **Файлы — канон, инструменты — тонкий слой.** Идеи cognee (semantic recall, lifecycle, граф) без его инфраструктуры.
- **Ничего не удаляется автоматически.** Lifecycle только помечает кандидатов; harvester переливает старое в архив, а не стирает.
- **Приватность.** Эмбеддинги считаются локально; наблюдения не ведутся в репозиториях из денилиста (`exclude_paths`); сам этот репозиторий — только код и шаблоны, никаких данных.
- **Токены — по факту работы.** Агенты просыпаются часто, но LLM зовут только когда реально накопилось (порог наблюдений, отлежавшиеся транскрипты).
- **Обратимость.** `uninstall.sh` снимает код и агенты, данные не трогает; `--purge-data` — отдельное осознанное решение.

## Ответ одним абзацем

Это самодельная долговременная память для Claude Code: обычные markdown-заметки плюс три фоновых процесса. Первый даёт семантический поиск по всей памяти — локально и бесплатно. Второй автоматически ведёт «дневник проектов»: новая сессия сама знает, где ты вчера остановился и был ли зелёным тестовый прогон. Третий выучивает твои привычки и на старте сессии напоминает их Claude — со временем его всё реже приходится поправлять. Всё лежит файлами, читается глазами и версионируется git'ом; облака и базы данных нет, LLM-токены тратятся копеечно и только по факту работы, любая часть выключается одной командой.
